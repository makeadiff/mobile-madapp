Frequently Asked Questions (FAQ)


1. What is the difference between MADApp and Mobile MADApp?
MADApp is a web portal to manage volunteer and student data in the MAD system. It is used to assign students, teachers and mentors to a particular class in a shelter. The data updation in MADApp is typically a one time activity at beginning of the year. While Mobile MADApp is a mobile compatible tool to enter qualitative class data on a week on week basis.
2. When I login to my Mobile MADApp, it says "No Classes Found"
- Ask the CH to ensure that he/she has done the batch allocation for all volunteers and students on MADApp
- Ask the Shelter Support Fellow (SSF) to ensure that a Class Start Date is assigned for the particular shelter. All classes that have already occurred post this date should appear on Mobile MADApp after this is done. Here is a tutorial for the SS Fellow to follow.

4. I seem to have forgotten my Mobile MADApp password, how do I reset it?
The password for MADApp and Mobile MADApp are always the same. By default, both tools have password set as pass. Mobile MADApp currently does not have a "forgot password" option. But you can reset the password using forgot password option of MADApp by checking this link
5. I cannot see the student names from my class in Mobile MADApp. What is wrong?
- It is possible that the student data does not exist on MADApp yet. Ask the CH to check whether he/she has updated the student data. Check this tutorial for ESF to add student and class data
- It is also possible that the student data might have been deleted for some reasons. Ask the CH to check in MADApp whether the student data is present or not.
6. How can I go about updating the details for my class in MADApp if the data is outdated or erroneous?
Ask the CH/ESF to make a change in MADApp. The changes should reflect within 24 hours
7. I cannot see the data I entered after my class last week anymore. Where did the data go?
This could happen due to two reasons:
- If Net connection was disrupted while entering and saving class details, the data might have been lost.
- If the volunteer forgot to click on the SAVE button at the end of the page after entering data, the data might have been lost.
A good practice for all volunteers after entering and saving data is to ensure that you receive the "Information Updated" notification. One can then be sure that data was saved properly
8. A teacher in my batch has left MAD but his/her name is still appearing in MADApp
Get in touch with the Human Capital fellow to ensure that volunteer exit status has been changed to alumni on MADApp.
